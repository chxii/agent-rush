## 1. 游戏概述与定位

### 1.1 一句话

> 你是一名 MEV 搜索者（searcher）——链上机会的猎手。你制定战略——挑机会、分资源、定预案、临场改令；再派出你的 **AI Executor Agent 自主地、长程地把战略执行下去**：它拆解任务、真实调用链上工具、观察结果，在被对手抢占或你改令时迭代修复，最终向你交付这一轮的战果。你出脑子，Executor 替你下场。

### 1.2 游戏类型与核心体验

Agent Rush 是一款**单人、过关式（roguelike）的策略卡牌游戏**，主题是以太坊上真实存在的 **MEV（Maximal Extractable Value，最大可提取价值）**——区块生产者和套利机器人通过对交易排序、抢先、夹击榨取的利润。

- **核心体验**：玩家扮演一名 MEV 搜索者（searcher），每一层面对一手"机会牌"（一次次链上套利机会），决定打哪些、各投多少 Gas、被抢时怎么办，然后交给自己的 AI Executor 去执行。玩家是出谋划策的"大脑"，Executor 是替你下场操作的唯一伙伴——是"你 + 你的 agent"，不是一支团队。
- **节奏**：闯关式，共 20 层，逐层加深。目标是走到终点并实现总盈利。
- **玩家张力来源**：有限 Gas 预算下的资源分配取舍 + 信息不对称（骗局牌） + 会越来越强的对手 + 执行中突发意外的临场应变。

### 1.3 目标玩家

- **首要**：Web3 新手 / 对 MEV 好奇但没接触过的人——游戏的教学目标是"玩过即懂 MEV"。
- **次要**：Hackathon 评委——关注 GLM 长程 Agent 能力的可见性与诚实性。
- **设计基调**：娱乐性教学游戏。用户的核心诉求是"**简化但不能教错**"——机制可以为可玩性做自觉简化，但不能给出与真实 MEV 经济学相悖的错误认知（详见 §14）。

### 1.4 核心卖点

- **真正的长程自主 Agent，不是念稿**：执行阶段由一个 LLM 驱动的 Executor 真实地"规划 → 调用工具 → 读取真实结果 → 遇意外重新规划 → 交付"。这是赛道要的长程能力，也是本作与"普通工作流 / 一次性 API 调用"的分水岭（详见 §13）。
- **玩家定战略、Agent 长程执行**：玩家不是旁观者，而是出谋划策的搜索者本人。玩家的每个决策（分 Gas、定预案、临场干预）都在给 Agent 制造"需要应变的环境"。
- **玩过即懂 MEV**：5 种牌型对应 5 种真实 MEV 策略，玩几关就能向别人解释什么是夹击、什么是抢跑。
- **零门槛**：浏览器打开即玩，URL 发出去评委就能上手。

### 1.5 设计支柱

| # | 支柱 | 含义 |
|---|------|------|
| P1 | 玩过即学会 | 不读文档也能理解 MEV、mempool、抢跑、夹击 |
| P2 | 压力即乐趣 | Gas 预算 + 对手竞争 + 信息不对称 = 强制做出取舍 |
| P3 | 可见的后果 | 每张牌的盈亏实时显示，Agent 的决策链完全透明 |
| P4 | 零门槛 | 无需钱包、合约、后端 |
| P5 | 长程能力可见 | Agent 的任务拆解、工具调用、迭代修复全过程可见，评委能指认 |

## 2. 核心循环：一局怎么玩

### 2.1 一局的宏观结构

一局游戏 = 闯过 20 层。**每一层是一个回合**，回合内分四个阶段顺序推进：

```
扫描  →  决策  →  执行  →  结算
(发牌)   (玩家)   (AI)    (报告)
```

打完一层结算后，若未触发失败条件，进入下一层（难度递增），直到通关（第 20 层且达标）或失败出局。**开局前先输入一个玩家 ID、再选一个起始角色**（见 §4），整局生效。

### 2.2 阶段一：扫描（系统发牌）

回合开始，系统按"当前场景"扫出一手**机会牌**——每张牌是一次 MEV 机会，带类型、稀有度、预期收益、Gas 成本、显示风险、真实风险、时间窗口、竞争度等属性（详见 §3）。

- 基础发牌数 = 3 张（`ROLE_CONFIG.base.scanCardCount`），上限 6 张。**侦察型角色**每档位额外 +1/2/3 张。
- 部分牌是**骗局牌**：显示风险极低（0.03~0.25）、实则真实风险极高（0.82~0.97），且利润被放大 1.5~2.5 倍引诱你下注。需要玩家自己识别——游戏**不再有"自动揭穿骗局"的能力**（原 RiskAnalyzer 已砍），识别骗局是玩家的核心职责。
- 教学关（1~3 层）发的是写死的固定牌组（见 §11），保证教学可复现。

### 2.3 阶段二：决策（玩家的主战场）

玩家制定一份**作战方案**，受 **Gas 预算上限**和**最大可同时执行牌数**双重约束：

1. **选哪些机会牌**——在有限预算和牌位里挑。每层最大选牌数随层数解锁：**1~2 层选 1 张 / 3~7 层选 2 张 / 8~20 层选 3 张**（`DECISION_LIMITS.maxSelectedCardsByLayer`）。
2. **给每张牌分配 Gas**——核心资源博弈。默认 Gas = 牌的 `gasCost`（GasRatio=1，无溢价加成）。投得越多成功率越高（封顶 2 倍 gasCost），但能打的牌就越少。几张牌**共享一个 Gas 池**，此消彼长。
3. **为每张牌设定遇袭预案**——预先决定"如果这张牌被对手抢占，怎么办"，默认 `fight`（硬刚）：
   - **硬刚 `fight`**：调 `replace_tx` 加价反抢（默认加到 1.2 倍 Gas），掷骰子看能否压制对手。
   - **放弃 `abandon`**：调 `abandon_card` 止损撤出，只烧 15% Gas。
   - **转移 `transfer`**：调 `scan_replacement` 找一张还没打、EV 为正的替代牌，找不到等于放弃。

> 取舍是真实的：Gas 全压一张牌 → 抗抢强但打不了几张；平均分 → 铺得开但每张都易被抢。没有无脑最优解。
> **选牌阶段计时**：非教学关默认 60 秒（`DEFAULT_PLAY_MS`，C4 从 25s 放宽）；教学关不计时，显示"练习中"。

### 2.4 阶段三：执行（AI Executor 的主场）

玩家把作战方案交给 **Executor**——一个由 LLM 驱动的自主 Agent。执行阶段是全游戏的 AI 核心，运作方式是**半闭环的"观察–思考–行动"循环**：

```
Executor 接到作战方案（选牌 + Gas 分配 + 每牌预案）
  └─ 初始规划：决定执行次序（含全部选中牌，只排序不弃牌）   [LLM]
  循环（逐张推进）：
    1. 对当前牌按工具序列调用工具（fetch_prices/monitor_mempool/broadcast_tx 等）
    2. 游戏侧工具模拟器执行，返回真实结果                  [确定性模拟器]
    3. 顺利 → 按既定计划走下一张（不打扰 LLM）
    4. 出意外（被抢/Gas不足/玩家干预）
       → 把当前真实战况丢回 LLM，重新规划剩余步骤          [LLM 重规划]
  └─ 全部处理完 → Executor 用人话交付结算报告              [LLM]
```

**为什么是"半闭环"而不是每步都问 LLM**：顺利执行时不打扰 LLM（省调用、省时间、不兜圈）；只有在意外发生时，才拿当前真实状态回头让 LLM 重新思考。意外时是"拿真实结果重想"，不是走预设分支——这正是长程、自主、迭代修复的体现。

**重规划的触发点**（`maxReplansPerRound: 4`，且 ≤ 牌数）：
- **被对手 Bot 抢占**（`target_stolen`）——核心高光点，可触发干预窗口与 `replace_tx` 反抢。
- **Gas 不足**（`gas_insufficient`）——剩余预算撑不起原计划。
- **玩家一次性干预**（`player_intervention`）。
- **交易失败 / 目标失效 / 窗口过期**（`tx_failed` / `target_invalid` / `window_expired`）——这些是**已结算的终结失败**，牌救不回；但它们仍会开启一个干预窗口，让玩家**重新调度剩余牌和 Gas**（C4 Round9 扩展，文案诚实区分"救回"与"重新调度"）。

> 注意：终结失败（`tx_failed` 等）**不再触发 LLM 重规划那张已败的牌**（C2 决策）——已结算的牌无可重规划，Executor 不会浪费调用。但保留了"重新调度剩余资源"的干预窗口。

**玩家的一次性实时干预**：每回合给玩家**一次**临场改令机会。仅当"干预有意义"时才弹出 **60 秒倒计时窗口**（带可收缩进度条 + 跳过按钮）：判定条件为事件是可处理类型、本回合干预未用过、且现场有 ≥1 张未结算牌 + 有可调度资源。提供三个快捷指令，或自然语言打字下令：
- **全部硬刚 `fight_all`**：把剩余 Gas 摊给所有还能动的牌。
- **放弃最高风险 `abandon_highest_risk`**：立刻止损风险最高的未结算牌。
- **Gas 集中最优 `focus_best_gas`**：把剩余 Gas 集中到当前 EV 最好的牌。

干预指令作为一条**高优先级观察**注入循环——Executor 当前动作完成后读到它，重新规划剩余动作。设计上刻意只给一次，避免"随时打字"的限时焦虑，让这一次改令成为关键决策。越早干预，可调整空间越大（打完的牌已锁定）。

**透明可见**：整个执行过程以"按牌分组的思维链面板"实时呈现——每张牌打到哪一步、调了什么工具、返回了什么、Executor 为什么这么决定，玩家全程看得见（P5）。执行顺序条（pipeline）显示 LLM 的真实执行序。

### 2.5 阶段四：结算（人话报告 + 输赢可见）

执行结束后，**"执行"按钮变为"结算"按钮，点击才弹出结算弹窗**（C4 改动，避免自动打断）。Executor 用人话总结这一轮：每张牌赚还是亏、为什么；本层净收益、累计收益、连亏进度；并实时显示距离胜利线和失败线还有多远。

未失败则进入下一层。Boss 层通关后弹出角色升级奖励（见 §4.3）。

## 3. 卡牌系统

### 3.1 机会牌的属性（`createCard`）

每张机会牌是一次 MEV 机会，包含：

| 属性 | 含义 | 怎么定的 |
|------|------|----------|
| `type` | 5 种 MEV 策略之一（见 §3.2） | 5 种等概率随机 |
| `rarity` | 稀有度 common/rare/epic/legendary | 按场景稀有度权重抽 |
| `expectedProfit` | 预期收益（ETH） | 按稀有度利润区间抽 |
| `gasCost` | 基础 Gas 成本 | 按稀有度 Gas 区间抽 |
| `displayedRisk` | 牌面展示的风险 | 正常牌 = trueRisk；骗局牌只显示 0.03~0.25 |
| `trueRisk` | 实际失败概率（结算用） | 按稀有度风险区间抽；骗局牌 0.82~0.97 |
| `isScam` | 是否骗局牌 | 按场景 `scamRate` 概率 |
| `competitionLevel` | 竞争激烈度（0~3） | 按稀有度，越稀有越多人抢 |
| `timeWindowSec` | 时间窗口（秒，10~50） | 随机；**目前只对清算牌的 `hardTimeWindow` 有名义作用，实际近乎氛围展示** |

**稀有度区间**（`RARITY`）：

| 稀有度 | 基础权重 | 利润区间 (ETH) | Gas 区间 | 风险区间 | 竞争度 |
|---|---|---|---|---|---|
| common 普通 | 55 | 0.05 ~ 0.5 | 15~40 | 0.05~0.45 | 0~1 |
| rare 稀有 | 30 | 0.3 ~ 1.5 | 35~70 | 0.2~0.6 | 1~2 |
| epic 史诗 | 12 | 1.0 ~ 3.0 | 60~120 | 0.4~0.8 | 2~3 |
| legendary 传说 | 3 | 3.0 ~ 8.0 | 80~150 | 0.55~0.9 | 3~3 |

> 读法：越稀有 → 利润越高，但风险越高、Gas 越贵、抢的人越多。没有"又肥又安全"的牌。
> （注：场景的稀有度权重会覆盖基础权重，见 §7。）

### 3.2 五种牌型（各有独特机制，对应真实 MEV 策略）

牌型不只是名字不同，每种类型在结算判定里有**独立的系数**（`CARD_TYPE_MECHANICS` + `CARD_TYPE_TOOL_SENSITIVITY`），导致打法完全不同：

| 牌型 | Gas成功权重 | 被抢系数 | 利润方差 | 特殊机制 | 性格 |
|---|---|---|---|---|---|
| **套利 arbitrage** | 0.75（低） | 1.1 | ±10% | 失败只烧 0.5× Gas（容错高） | 稳健型。堆 Gas 提成功率弱，但失败损失小、收益稳。人人看得见、抢的人多、利润被磨薄 |
| **夹击 sandwich** | 1.75（最高） | 1.25 | ±30% | — | 时序型。**最吃 Gas**，砸钱显著提成功率，但收益波动大 |
| **NFT 抢购 nft_snipe** | 0.9 | 1.65（最易被抢） | ±45% | — | 高波动型。最容易被抢、收益极不稳、窗口短，最刺激也最易扑空 |
| **抢跑 front_run** | 1.15 | 1.2 | ±18% | 出价博弈：高过对手 +28%、低于 −34%；反抢阈值 1.1 | 对抗型。纯比 Gas 出价是否压过对手 |
| **清算 liquidation** | 0.85（低） | 0.9（不易被抢） | ±5% | `hardTimeWindow: true`（名义） | 窗口型。收益最稳、不易被抢，但堆 Gas 提成功率效果弱，靠选低风险牌取胜 |

> **工具敏感度**（影响各关卡判定）：套利 price 1.25 / 夹击 mempool 1.25 / 抢跑 mempool 1.35 / NFT price 1.15。
> 这 5 种对应真实以太坊 MEV 的主要收益来源（详见 §14）。

### 3.3 各牌型的工具调用序列（`CARD_TOOL_SEQUENCES`）

执行时每张牌按其类型的工具序列推进：

- 套利 / NFT抢购 / 清算：`fetch_prices → monitor_mempool → broadcast_tx`
- 夹击 / 抢跑：`monitor_mempool → broadcast_tx`（无价格查询，直接侦察后广播）
- 默认：`monitor_mempool → broadcast_tx`

### 3.4 骗局牌（scam）—— 游戏最核心的陷阱

- 伪装成超肥的低风险牌：`displayedRisk` 只显示 0.03~0.25（看着超安全），`trueRisk` 实际 0.82~0.97（几乎必失败），利润还被放大 1.5~2.5 倍。
- 结算按 `trueRisk` 算，一旦下注几乎血本无归（烧 Gas）。
- **Executor 也会被骗**：喂给 LLM/decider 的牌数据**只含 `displayedRisk`，剔除了 `trueRisk` 和 `isScam`**（C4 Round6 修复的信息泄漏）。Executor 和玩家信息对等，识别骗局是**玩家的职责**，别指望 AI 帮你避开。
- 这是"多看牌"（侦察型角色）价值的根源：牌发得越多，越能挑掉伪装牌。

### 3.5 牌间关联（互斥牌）—— 已 mock，未来扩展

v6 设计里"互斥牌（conflict）"是中后期复杂度来源。**实际实现中已降级为 mock，当前代码无互斥逻辑**（全仓 grep `conflict`/`mutex` 零命中）。逐张独立结算。未来若做 B2 阶段再补，届时应把 sandwich、liquidation 纳入互斥（C1 审计建议）。组合牌（combo）同样作为后期扩展保留，不在当前版本。

## 4. 角色系统（起始角色）

### 4.1 开局三选一

开局从 3 个风格化角色中选一个，提供**透明的规则型 buff**，整局生效。buff 规则公开告诉玩家——玩家知道自己的底色，才能围绕它做策略。这比"假装是 AI"更诚实，也更有深度。每个角色只强化**一个维度**，其余维度走基础值（首版纯正向 buff，无副作用）。角色最高 3 档（`maxRoleLevel: 3`）。

| 角色 | 风格 | buff（Lv1/2/3） | 挂点机制 |
|------|------|----------------|----------|
| **🔭 侦察型 scout** | 信息优势 | 每轮额外发 **+1/+2/+3** 张机会牌 | `CardGenerator.runSearcher` 发牌数 |
| **🛡️ 抗压型 resist** | 对抗韧性 | 被抢概率系数降至 **0.7/0.55/0.45**；反抢所需出价倍数降至 **0.84/0.76/0.68**、压制率额外 **+0.08/0.13/0.18** | `ToolSimulator` 抢占判定 + `replace_tx` |
| **⚡ 效率型 efficiency** | 资源优势 | 每层 Gas 池上限 **×1.2/1.35/1.5** | `GameState.gasPoolMaxForStage` |

> 角色 buff 只是给已有机制加修正系数，不引入新系统，工程可控。
> 三角色强度接近，靠风格区分而非数值碾压（C2 实测三角色通关率差异 6.9pp，目标 ≤10pp）。

### 4.2 被砍掉的旧能力

v6/早期设计里的另外两个"规则 Agent"在 B1 阶段被砍：
- **骗局揭示（原 RiskAnalyzer）**：砍掉后骗局牌保持伪装值，玩家得自己识别——这正是让"风险"重新成为真考验的目的。（Backlog：未来可做成"教学层限定"帮新手，之后消失。）
- **推荐高亮（原 Strategist）**：替玩家做决策、削弱核心循环，且原本就是无调用点的死代码，直接删除。

### 4.3 成长系统：Boss 层升级角色档位

- 角色开局选定、整局不变。
- **Boss 层通关奖励 = 强化当前角色的能力档位**（roleLevel +1，最高 Lv3）。取代了旧版"逐层解锁新 Agent"的设计。
- Boss 层位于 **7 / 12 / 17 / 20 层**（见 §7）。前三个 Boss（7/12/17）各升一档，到第 17 层后角色满级 Lv3。
- 旧的"上阵名额 slots / 逐层解锁 unlocks / activeAgents"状态体系已在 B1 全部移除。

### 4.4 Executor 不是可选角色

Executor 是**固定主角**，执行阶段从第 1 层起就由它驱动，不需要解锁、不占阵容名额、不参与角色 buff 体系。它是全游戏唯一的 LLM 核心（详见 §5）。原"刚体模式 vs 自适应模式"二分已取消，执行阶段始终是半闭环。

## 5. Executor（AI）与对手 Bot

本作诚实区分：**玩家侧的 Executor 是真 LLM Agent，对手是规则驱动的 Bot**，不为了"看起来都是 AI"而堆叠。

### 5.1 Executor —— 唯一的 AI 核心

- **从第 1 层起就在场**，每一局、每个玩家、执行阶段都由它驱动。把游戏唯一的 AI 亮点从头展示。
- **职责（LLM 擅长、工作流不可替代的部分）**：初始规划（执行次序）、迭代修复（遇袭 / 干预时结合真实状态重新决策）、交付解释（人话总结）。
- **不负责的部分（交给游戏侧，必须可靠）**：所有数值结算（盈亏、Gas 扣减、成败判定）由工具模拟器算，LLM 绝不直接编造数值。**LLM 决定"做什么"，模拟器决定"结果如何"。**
- **信息边界**：LLM 拿到的是展示口径（含 `displayedRisk`，不含 `trueRisk`/`isScam`），靠当前局面的真实数字现场判断，会和玩家一样被骗局牌骗。
- **system prompt 只管"怎么答"**（返回严格 JSON、简洁中文、保留 MEV 术语英文、Gas 由玩家定不要自己编），几乎不含游戏规则；**user prompt 管"答什么"**（callType + schema + 整个局面 `JSON.stringify(input)`）。`temperature: 0.2`。

### 5.2 三种执行后端（取决于运行环境）

游戏实际有三条执行路径，对玩家透明但口径很重要：

| 条件 | 后端 | 行为 |
|------|------|------|
| 有 API key / 代理模式（`isProxyMode()`） | **LLMDecider**（真 GLM） | LLM 看牌自行排序，大方向"价值高的先打"但**有裁量、非严格 EV 序、每次可能略不同** |
| 无 key | **RuleDecider**（确定性保底） | **严格按 EV 降序**（EV = 利润×(1−trueRisk) − Gas） |
| 教学前三关 | **tutorialLayerDecider**（脚本化） | 包一层 RuleDecider，**全程零 LLM 调用**，结算文案按关定制 |

> **执行顺序文案口径**：统一用"优先打更有价值的牌"，**不要写死"严格按 EV 排序"**（只对 RuleDecider 成立）。玩家选牌顺序不影响执行顺序，只能靠预案 / 干预影响。

### 5.3 健壮性铁律与降级

- 任何 LLM 返回都做字段校验（`SchemaValidator`），缺字段即降级，残缺数据绝不流进核心逻辑。
- LLM 不可用 / 返回不合法 / 流式停滞 / 超时（请求 20s、流式读取 15s）→ 退到 RuleDecider 保底，但思维链面板打 `[自动保底]` 标注，不假装是 AI 在思考。
- 重规划次数达上限（4 次）也切 RuleDecider。
- 相同局面（层数+牌+事件）会缓存，不重复调用。
- `executionOrder` 必含全部选中牌（提示词收紧"只排序不弃牌" + `normalizeInitialPlan` 把漏排牌补回末尾），消除"LLM 说放弃却照样执行"的矛盾。

### 5.4 工具模拟器 —— Executor 的"真实世界"

工具是游戏规则的唯一出口，由游戏侧实现为**确定性的链上行为模拟器**，吃 Executor 给的参数、吐真实且影响后续的结果（详细数值见 §6）：

| 工具 | 做什么 | 返回的关键结果 |
|------|--------|----------------|
| `fetch_prices` | 读当前行情 / 价差 | 价差是否还在、是否仍有利可图、滑点估计 |
| `monitor_mempool` | 侦察竞争态势 | 是否有对手盯上、对手出价档位、抢占概率 |
| `broadcast_tx` | 提交交易，真正"打这张牌" | 成交 / 失败、实际收益、实际 Gas 消耗、是否被抢 |
| `replace_tx` | 抬价替换在途交易（反抢） | 是否压制成功、新 Gas 消耗、剩余预算 |
| `scan_replacement` | 找替代机会（原目标黄了时） | 是否找到替代、替代牌概要 |
| `reallocate_gas` | 牌之间重分剩余 Gas | 更新后各牌 Gas、剩余池 |
| `abandon_card` | 放弃某牌止损 | 确认放弃、退回 / 损失的 Gas |

**受控随机 + 种子开关**：结果由牌的真实属性（`trueRisk`、`competitionLevel`）、对手强度、已投 Gas 共同决定，随机只在受控范围内（收益 ±方差浮动）。玩家能算"期望"，但单次有方差——这是 Agent 应变"有戏可演"的前提。保留固定随机种子开关，Demo / 测试时可切为可复现（`createRandomSource(seed)`）。

### 5.5 对手 Bot —— 规则驱动

对手是 Bot（规则 AI，**不是** LLM），五个 Bot 按层数登场，基础抢夺率递增：

| Bot        | 基础强度 | Gas 池 | 登场层               |
| ---------- | ---- | ----- | ----------------- |
| 🐣 Bot-404 | 0.17 | 100   | 1~3（教学，仅第3关登场抢一次） |
| Shadow     | 0.32 | 180   | 4~7               |
| Phantom    | 0.55 | 250   | 8~12              |
| Phantom+   | 0.75 | 300   | 13~17             |
| Genesis    | 0.95 | 350   | 18~20（终局）         |

- **Boss 层加难**：Boss 层（7/12/17/20）对手强度 `+0.07`（`boss.botStrengthBonus`），Boss 真的更难，不只是发奖励。
- **Genesis 专属机制覆盖**：压制率上限降到 0.6、对手竞价倍数提到 1.3（`botMechanicOverrides.Genesis`）——更难反抢、出价更凶。

### 5.6 对手学习 —— 已 mock，未来扩展

v6 设计的核心加分点是"对手会学习你的牌型偏好并针对性反制"。**实际实现中这是 mock**：`GameState` 保留了 `genesisHistory: { lastTwoRounds, boostedType }` 字段，但代码注释明确写 "Reserved for a future enemy-learning mock. B3 has no runtime read/write logic for it."——**当前没有任何运行时读写逻辑，对手不会真的学习**。对手强度纯靠层数 + Boss 加成递增。"双向适应"叙事是未来扩展方向，当前不成立。

## 6. 工具模拟器与结算数值

一张选中的牌按四道关卡结算（`ToolSimulator.js` + `toolSimulator.js` 配置）。任何一道没过就失败。

### 关卡 1：机会还在吗？（`fetch_prices`）
```
失效率 = (0.03 + trueRisk × 0.18) × 牌型价格敏感度
```
没过 → 判失败（机会失效 `invalidOpportunity`），烧 **8%** Gas。

### 关卡 2：会被对手抢吗？（`monitor_mempool` + 广播时判）
```
被抢率 = clamp01( (对手强度 × 0.45 + 竞争度 × 0.12 − 你的Gas × 0.003)
                  × 牌型mempool敏感度 × 牌型被抢系数 × 角色抗抢系数 )
```
- 对手越强、竞争度越高 → 越易被抢；Gas 防抢权重仅 0.003（单靠堆 Gas 防抢很低效）。
- 抗压型把角色抗抢系数降到 0.7/0.55/0.45，是真正的防抢手段。
- 被抢 → 失败，烧 **35%** Gas（可触发预案 / 干预反抢）。
- 检测基础值 `detectionBase: 0.145`；对手竞价 = 你的 Gas × 1.12（Genesis 1.3），最低抬价 8 Gas。

### 关卡 3：清算牌专属时间窗口
只有清算牌 `hardTimeWindow: true`。模拟耗时（`simulatedToolElapsedSec: 1`）恒小于窗口值，**这道关基本不触发**——时间窗口属性近乎氛围展示（已知简化，Hackathon 后再做）。

### 关卡 4：交易成功吗？（`broadcast_tx`，最关键）
```
成功率 = clamp(
  [ 0.58                                    ← baseSuccessProbability
    + min(GasRatio, 2) × 0.18 × 牌型Gas成功权重  ← Gas 加成（gasWeight）
    − trueRisk × 0.85                        ← riskPenalty
    − 对手强度 × 0.24 ]                       ← botPressurePenalty
  × 牌型广播敏感度,
  下限 0.05, 上限 0.9 )

其中 GasRatio = 你给的Gas / 牌的gasCost
```
- 成功 → 拿 `expectedProfit × (1 ± 牌型方差)`（夹击±30%、清算±5%，等等）。
- 失败 → 烧 **40%** Gas（`txFailed`）。

### 失败烧 Gas 比率（按原因，`failedGasLossRateByReason`）

| 失败原因 | 烧 Gas 比率 |
|---------|------------|
| `invalidOpportunity` 机会失效 | 8% |
| `windowExpired` 窗口过期 | 12% |
| `stolen` 被抢 | 35% |
| `txFailed` 交易失败 | 40% |
| `abandon` 主动放弃 | 15% |
| 套利牌失败 | 额外 ×0.5（容错） |

> C1/C2 修正：早期 `failedGasLossRate=1`（失败全损）与真实方向相反，已降为按原因分类（最高 40%）。

### 反抢（硬刚）压制率（`replace_tx`）
```
压制率 = clamp( 0.35 + 出价优势 × 1.5 + 角色抗抢加成, 0, 上限 0.9 )
要求：新Gas ≥ 对手出价 × 要求倍数（默认 1.05；抢跑牌 1.1）
```
抗压型让要求倍数降到 0.84/0.76/0.68、压制率额外 +0.08/0.13/0.18——抗压型配硬刚是强组合。

### EV 决策参考公式（UI 展示）
```
EV = 预期利润 × (1 − 真实风险) − Gas × 0.001
```
这是给玩家横向比牌的参考值，**不是单次结果**。单次实际收益因"成败掷骰 + 成功后浮动 + 失败烧 Gas"会偏离 EV。

### 半闭环预算（`SEMI_LOOP_CONFIG`）
- 每回合最多重规划 4 次（且 ≤ 牌数）。
- 工具间延迟 `toolDelayMs: 3500`（正常关；教学第3关另用 600~900ms 放慢演示）。
- 模拟工具耗时 1 秒。

## 9. 输赢条件（`WIN_LOSS_CONFIG`）

| | 条件 | 设计原则 |
|---|------|----------|
| **胜利** | 第 20 层 **且** 累计收益 > **8.75 ETH** | 走到终点 + 总账真正盈利 |
| **失败** | 连亏 ≥ **2** **且** 累计收益 < **−0.5 ETH**（双条件同时） | 给"翻盘缓冲"，一两次亏损不出局 |
| **第 20 层判负** | 到第 20 层但累计 ≤ 8.75 → 判负 | 堵掉"卡在第 20 层无限重打凑数"的旧漏洞 |

**连亏计数规则**（`applyRoundResult`）：净收益 <0 → 连亏 +1；>0 → 清零；**==0（打平）→ 不清零也不增加**。

**跳过躺赢漏洞修复**：空作战方案（跳过）会被记为 **−0.1 ETH/层**（`applySkipPenalty`），不再是零成本免死金牌。连续跳过会拖低累计收益、触发失败条件。胜利 / 失败阈值本身保持 C2 锁定值未动。

**硬性 UI 要求**：顶部控制台实时展示距离胜利线和失败线还有多远——"不知道怎么赢/输"是旧版最大缺陷。逼近失败阈值时"连续失败"数字转危险红。

## 10. 难度与复杂度曲线

核心原则：**逐层解锁复杂度，每一关只新增一个需要理解的概念**。三条难度线错峰上升：

| 阶段 | 复杂度引入 | 对手 | 玩家学习重点 |
|------|-----------|------|--------------|
| 教学（1~3） | 单张牌、5 种牌型、骗局牌、Gas/EV、预案/干预 | Bot-404（仅第3关抢一次） | 看懂牌、配 Gas、应对被抢 |
| 前期（4~7） | 多场景选择、对手开始竞争 | Shadow（Boss 7） | 选猎场、配预案 |
| 中期（8~12） | 资源压力上升 | Phantom（Boss 12） | 牌的取舍、抗抢 |
| 后期（13~17） | 高竞争、四场景混选 | Phantom+（Boss 17） | 临场应变 |
| 终局（18~20） | 全要素、最强对手 | Genesis（Boss 20） | 综合博弈 |

三条线：**对手强度**（抢夺率随层递增 + Boss +0.07 + Genesis 专属机制）、**局面复杂度**（场景数从 1 增到 4、可选牌数 1→2→3）、**资源压力**（Gas 池递增但牌成本与数量增长更快）。任一关只新增一个难度来源，避免陡增劝退。

> 注：v6 设想的"互斥牌"和"对手学习"作为中后期复杂度来源**均已 mock，当前不生效**（见 §3.5、§5.6）。当前难度主要靠对手强度 + 资源压力 + 信息不对称（骗局率随场景升高）。

## 11. 教学系统（前三关）

前三关是**全程脚本化、零 LLM 调用**的交互式教学（`tutorialLayerDecider`），手牌写死保证可复现，不计时（显示"练习中"）。右栏 terminal 逐条流式推送 Executor 第二人称带教叙事（间隔 1200ms，第3关执行段 600~900ms），左栏给结构化数据。教学只对未通关过的玩家生效（`isActiveTutorial() = isTutorial && !tutorialSeen`）；老玩家走正常随机发牌。调试用 `?tutorial=1` 强制走教学。

### 第 1 关 — 先排雷，再认牌
- **目标**：识别骗局牌 + 认全 5 种牌型。
- **牌组**：6 张写死牌，含 1 张骗局牌（伪装 epic 套利，显示风险 0.08 / 真实 0.92 / 利润 3.8）+ 5 种正经牌各一。
- **交互**：玩家首次点到每种类型，才推送该类型一句话说明（按 type 去重）。
- **结果按真实结算**（掷骰，正经牌也可能失败）。点评三分支：选骗局牌→烧光 Gas / 选正经且成功→"干净利落" / **选正经但失败→安抚"你没选错，是概率没掷过，只亏少量 Gas"**。
- **bot**：Bot-404 本关不抢，纯练手。失败也放行进下一关。

### 第 2 关 — 配 Gas，看 EV
- **目标**：Gas 池 = 每层预算（用不完不扣、不结转）+ 调 Gas 看成功率/EV 变化 + 理解 EV 是平均值。
- **牌组**：5 种牌型各一（sandwich 为推荐解），选牌上限 1 张。实时显示每张牌 EV + 成功率。
- **关键讲解**："成功率 vs EV 公式数字为何不同"——EV 用稳定的 trueRisk 基准、成功率随 Gas/竞争/Bot 实时变，诚实标注是游戏刻意拆成两个旋钮的简化。引出博弈三要素。
- **bot**：不登场（"下一关有人来抢"作钩子）。

### 第 3 关 — 预案与干预（核心）
- **目标**：理解三预案 + 看懂完整执行流程（Executor 自排序）+ 亲历"被抢→干预"并真实看到干预改变结果 + 理解干预时机。
- **关键机制**：本关**可选 2 张牌**（只有 2 张牌干预的 Gas 分配/集中/放弃才有意义）；**按真实结算跑**（不写死结果），仅手牌 + 被抢事件写死。
- **流程**：
  1. 选牌阶段：介绍 Bot-404 上线要抢、可选 2 张共用 Gas 池、给每张设预案（左栏预案表）。
  2. 执行流程讲解（点执行后、被抢前）：Executor 说明"执行顺序是我排的、默认优先打更有价值的牌"。
  3. **被抢→暂停→干预**：脚本在第一张牌广播时触发被抢（`createForceStealHook`），执行暂停、第二张牌仍 pending。Executor 推送"被抢了/还有一张能改命运"，讲三个快捷键。
     - 点快捷键 → 按 RuleDecider 真实解析回话。
     - 打字下令 → placeholder 给例子，**诚实告知教学关脚本化不真解析，真实游戏接 AI 后会读懂**，回显原文 + 说明。
     - 不干预则一直暂停（本关唯一卡点），约 10 秒催一句。
  4. 执行继续，按真实结算跑完进结算。
- **收尾**：按真实 netProfit 二选一点评 → 总结 Executor 是 AI Agent → "三课学完" → 引向真实战场。

> **可选牌数平滑过渡**：1~2 关 1 张 → 3 关 2 张 → 4~7 关 2 张 → 8~20 关 3 张。
> **新手引导层（页面 tooltip/coachmark）= 不做**：靠交互式教学 + demo/口播，最多首回合加几条静态行内灰字。

## 13. AI 必要性说明（给评委的诚实技术叙事）

评委最会问："这个用普通工作流也能做，为什么需要 LLM？" 回答是**诚实地划清边界**：

**全游戏只有一个地方用 LLM——Executor 的执行循环——但那个地方工作流替代不了：**

1. **环境是持续变化、对抗性的，不是预设分支。** 受控随机让意外真的会突发（被抢、失败、Gas 不足），Boss 层对手更强。Executor 面对的不是"if 被抢 then 走分支 B"，而是拿当前真实战况（剩余 Gas、各牌实际盈亏、玩家预案、对手出价）重新规划剩余步骤。
2. **它真的在读工具的真实返回再决策**，不是念预排好的稿。工具返回会改变它的下一步——这是"自主持续执行"与"普通 API 调用"的分水岭。
3. **玩家的一次性自然语言干预**需要 LLM 解读意图并融入剩余规划，是规则分支难以覆盖的开放输入。

**反过来，诚实承认哪些不是 AI：**
- 对手 Bot 是规则驱动的（按层数 + Boss 加成调强度），不是 LLM；对手学习功能当前是 mock，未实现。
- 角色 buff 是透明规则修正，不是 LLM。
- 所有数值结算由确定性模拟器算，LLM 不碰。
- 教学前三关是脚本化的（无 LLM 调用），第3关的自定义干预不真解析——已诚实告知玩家。

这种"AI 集中在一个不可替代的点、其余诚实标注为规则"的设计，比"每个角色都假装是 AI"更经得起追问，也更凸显 GLM 的长程能力。

## 14. 现实性说明（机制 vs 真实 MEV，2024–2026）

经 C1 真实性审计（联网核实一手资料），结论：**游戏建模方向正确，是对早期 PGA 时代 MEV 的合理简化，无原则性错误**，仅有一处时代背景需诚实标注。

### 14.1 五种牌型的真实原型

| 牌型 | 真实原型 | 准确性 |
|---|---|---|
| 套利 arbitrage | 原子套利，PGA 发源地、抢占最激烈 | C1 曾指出游戏早期把它设成"最不易被抢"方向反了，C2 已修正：保留稳健人设，但"稳"重释为失败容错高（×0.5 烧 Gas，对应 revert protection），被抢系数提到 1.1 |
| 夹击 sandwich | 前跑-受害-后跑同区块相邻，排序约束最硬 | 文案不再宣称时序宽松；方差已拉到 ±30% |
| 抢跑 front_run | 出价压过对手（priority fee） | 文案口径=给构建者的 priority fee/bundle 出价；反抢阈值 1.1 |
| 清算 liquidation | 状态触发（HF<1 可清算），机会因被抢先/价格回弹消失 | "时间窗口"注明是俗称（真实非倒计时）；状态触发模型留未来 |
| NFT 抢购 nft_snipe | 抢 mint 名额/低价挂单 | 注明是广义 MEV，区别于核心 MEV |

### 14.2 对得上现实的部分
- 高 Gas → 更可能抢到（PGA 竞价）、机会会消失/被抢、实际收益有浮动、EV 为正才长期值得做。
- 玩家 = searcher（抢机会的搜索者），"被抢" = 自己竞争失败 revert，**不是被夹的受害者**。
- `replace_tx` 抬价反抢 = 真实的"优先费竞价"。

### 14.3 最重要的时代性简化（务必让玩家知道）
游戏建模的是 **2019–2021 年经典 MEV**：公开 mempool 里逐笔 gas 公开竞价（大众心智模型）。**当下主流已转向 Flashbots / PBS（提议者-构建者分离）/ 私有订单流**：
- 现代 Flashbots bundle 竞价是**密封的**（搬进了 builder 的"小黑屋"）；
- **失败的 bundle 不上链、不烧 gas**——这恰和游戏"失败烧 Gas"相反。

| 游戏概念 | 真实世界现在 | 是否成立 |
|---|---|---|
| MEV 本身 | 一年仍抽几十亿美元 | ✅ |
| 5 种牌型 | 全都还在，占比没变 | ✅ |
| 靠出价抢机会 | 还在抢，从公开喊价变密封投标 | ✅ 行为成立、场所变了 |
| `monitor_mempool` 侦察对手 | 公开池里对手变少（藏进 builder/私有池） | ⚠️ 唯一偏旧的点 |
| 失败烧 Gas | 符早期 PGA，不符现代 Flashbots | ⚠️ 时代性简化 |

**为什么仍选旧模型（自觉简化）**：① 它是大众心智模型，最快让人懂 MEV；② 密封拍卖没法玩（看不到对手，对抗紧张感全没）；③ 诚实标注就站得住。规则页免责声明已点明"失败烧 Gas 仅符早期 PGA"。

> 来源：Figment《Brief History of MEV》、Flash Boys 2.0（Daian 2019, arxiv 1904.05234）、Flashbots Docs。
> 骗局牌是游戏化教学隐喻（蜜罐/钓鱼），非真实机制；具体数字（0.58/0.85 等）是平衡值非现实参数。

  - `src/ui/` — 渲染层（UIRenderer/ThoughtChainPanel/OverlayManager/SettlementPanel）。

- **模型**：GLM（Z.ai coding plan，OpenAI 兼容）。`GLM_BASE_URL=https://api.z.ai/api/coding/paas/v4`、`GLM_MODEL=glm-5.1`。

### 16.3 线上部署（Vercel 薄代理，v1.0.0 已上线）

- **`api/llm.js`**（Vercel Edge serverless）：从环境变量注入 key 转发到 GLM endpoint，SSE 原样 pipe，主动 delete body 里的 key，错误透传。
- **限流**：纯 IP 内存桶 90 次/分 + 700 次/天，预留 `checkAccess()` 钩子（日后加口令改一处）。

## 18. 已知风险点

1. **expert 档通关率卡区间下限**（~82%，目标 80~92%）：后续若叠加难度需重验，避免高手也被劝退。
2. **时间窗口机制近乎失效**：清算牌 `hardTimeWindow` 因模拟耗时恒小于窗口基本不触发，目前是氛围展示。若玩家深究会发现"时间窗口没用"。
3. **LLM 依赖与成本**：真实 LLM 流式有延迟与翻车风险；Mock 兜底可保跑完但失去 AI 展示。Demo 建议固定种子彩排 happy path。
4. **线上限流**：IP 内存桶限流（90/分、700/天）在 serverless 多实例下不共享，限流是软约束；Hackathon 后需轮换 key。
5. **时代性简化的认知风险**："失败烧 Gas"与现代 Flashbots 相反，已在规则页标注，但仍可能被熟悉现状的评委质疑——靠诚实叙事兜。
6. **对手学习是 mock**："双向适应"曾是 v6 核心卖点，当前不成立，对外叙事不应宣称对手会学习。
7. **教学第3关脚本化**：被抢事件写死、自定义干预不真解析——已诚实告知，但若评委在教学关试图测真实 AI 能力会落空（应引导他们玩第4层起的真实关卡）。

## 19. 概念词汇表

- **MEV（最大可提取价值）**：通过对链上交易的选择、排除、重新排序所能榨取的利润。本作主题。
- **searcher（搜索者）**：玩家的真实身份——抢链上机会的人。
- **机会牌**：一次 MEV 机会，有类型、稀有度、收益、Gas 成本、显示/真实风险、时间窗口、竞争度。
- **Gas / Gas 池 / priority fee**：链上手续费即 Gas（插队费）；每层一个 Gas 池（预算），用不完不结转；真正决定排序的是 priority fee。
- **显示风险 vs 真实风险**：骗局牌伪装成低风险（显示 0.03~0.25），真实 0.82~0.97。
- **5 种牌型**：套利/夹击/NFT抢购/抢跑/清算，各有独特机制（§3.2）。
- **遇袭预案**：为每张牌预设的被抢应对（硬刚/放弃/转移）。
- **Executor**：固定主角，执行阶段的 LLM 半闭环 Agent，无需解锁。
- **工具模拟器**：游戏侧确定性的链上行为模拟器，是 Executor 调工具的真实结果来源，也是所有数值结算的唯一出口。
- **半闭环**：Executor 顺利时按计划走、不打扰 LLM，只在意外时拿真实状态回头让 LLM 重规划的执行模式。
- **一次性干预**：执行中玩家的一次临场改令（每回合限一次，60s 窗口）。
- **对手 Bot**：规则驱动的竞争者（Bot-404/Shadow/Phantom/Phantom+/Genesis），会抢占机会牌。
- **起始角色**：开局三选一（侦察/抗压/效率），透明规则 buff，整局生效。
- **Boss 层**：第 7/12/17/20 层，对手加难，通关升角色档位。
- **回合阶段**：扫描 → 决策 → 执行 → 结算。
- **mempool**：交易公共候车区，对应 `monitor_mempool` 工具。
- **EV（预期价值）**：利润×(1−真实风险) − Gas×系数，横向比牌的参考值，非单次结果。
